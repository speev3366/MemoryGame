# Memory Duel V7.2.1

Тази версия добавя:
- по-стабилен auth поток със Supabase
- регистрация и вход с валидация на полетата
- профил панел
- история на мачовете, която се пази в базата
- вътрешен скрол в онлайн/auth панела, за да не излизат елементи извън екрана

## Ако проектът е нов
1. Отвори `supabase-schema.sql` в Supabase SQL Editor.
2. Натисни `Run`.
3. Попълни `supabase-config.js`.

## Ако вече имаш стар Supabase проект
1. Отвори `supabase-migration-v7.2.1.sql`.
2. Копирай съдържанието му в Supabase SQL Editor.
3. Натисни `Run`.

Тази migration добавя таблицата `match_history`, от която профилът чете историята.

## Как да го качиш в GitHub Pages
1. Качи всички файлове от папката в root на repository-то.
2. Увери се, че `index.html` е на най-горното ниво.
3. В `Settings -> Pages` избери `Deploy from branch`, `main`, `/(root)`.
4. Изчакай deploy-а да приключи.

## Онлайн поток
- Избираш `Онлайн`
- Влизаш или се регистрираш
- Създаваш стая или влизаш с код
- Домакинът дава `Старт онлайн`
- След края на мача историята се записва в `match_history`

## Важно
- Локалният режим и режимът срещу компютър работят и без профил.
- Историята в базата се записва само ако има активен профил.
- GitHub Pages е само за фронтенда; Supabase е backend-ът.
